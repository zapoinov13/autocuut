## Что не так сейчас

- B-roll подбираются "от балды" — AI видит только сухой текст блока без контекста темы видео и часто выдаёт расплывчатый запрос ("people talking", "city").
- Pexels берёт первое попавшееся видео без проверки длительности — часто короче блока, поэтому "дёргается".
- В UI можно только запустить авто-подбор целиком. Нет способа: заменить клип в одном блоке, удалить только его, поменять поисковый запрос, посмотреть альтернативы, загрузить свой файл прямо в блок.
- Длительность блока не контролируется при поиске (`min_duration` Pexels не используется).
- Нет понятия "тема видео" — AI не знает, о чём вообще ролик.

## Что сделаем

### 1. Edge function `fetch-broll` — умнее подбор

- Передавать AI **общую тему ролика** (заголовок проекта + первые 2 сцены как контекст) — он перестанет промахиваться по смыслу.
- Просить AI выдать **3 варианта запроса** на блок (primary + 2 fallback), а не один.
- Pexels: запрашивать `min_duration = block_seconds`, брать первое HD видео нужной ориентации, длиной ≥ блока. Если не нашли — пробовать fallback-запросы.
- Сохранять в БД не только URL, но и **метаданные подбора** (query, reason, alternatives) — чтобы UI мог показать и дать "перевыбрать".

### 2. Новая edge function `broll-block-action`

Точечные операции над одним блоком (не трогая остальные):
- `regenerate` — взять следующий вариант из alternatives или новый Pexels-результат по тому же запросу.
- `search` — пользователь вводит свой запрос, функция ищет на Pexels и возвращает 6 превью.
- `pick` — пользователь выбирает конкретный URL из превью → проставляется на все сцены блока.
- `clear` — снять B-roll с блока.

### 3. Хранение метаданных блоков

Новая колонка `scenes.broll_meta jsonb` — `{ block_id, query, reason, alternatives: string[] }`. Все сцены одного блока получают одинаковый `block_id`, чтобы UI группировал и операции применялись на блок целиком.

### 4. Переработка `BrollPanel.tsx`

Каждая карточка блока в отчёте становится **интерактивной**:

```
┌─ 00:12—00:18 · 6с · [Применён]  ──────────┐
│ "...врачи в Германии используют..."        │
│ «Конкретное место + действие»              │
│ ▶ превью видео (hover = play, mute)        │
│                                            │
│ [↻ Другой вариант] [🔍 Свой запрос]         │
│ [📁 Загрузить файл] [✕ Убрать]              │
└────────────────────────────────────────────┘
```

При клике "Свой запрос" — появляется input + сетка 6 превью Pexels, клик по превью применяет.
Для блоков `skipped_*` — кнопка "Добавить B-roll сюда" (открывает тот же поиск).

### 5. Точечное редактирование в `ScenesPanel.tsx`

Чип B-roll у сцены: добавить превью при наведении и кнопку "Заменить" рядом с "Удалить" — открывает мини-поиск Pexels.

## Технические детали

**Миграция БД:**
```sql
ALTER TABLE public.scenes ADD COLUMN broll_meta jsonb;
```

**Файлы:**
- `supabase/functions/fetch-broll/index.ts` — добавить тему, alternatives, min_duration, запись broll_meta + block_id.
- `supabase/functions/broll-block-action/index.ts` — новая, действия regenerate/search/pick/clear.
- `src/components/editor/panels/BrollPanel.tsx` — интерактивные карточки блоков, поиск Pexels внутри панели.
- `src/components/editor/panels/ScenesPanel.tsx` — кнопка "Заменить" + превью у чипа B-roll.

**Важно:** все операции block-action идут через service role (RLS check по `user_id` из JWT), не дают править чужие проекты.

## Порядок работы

1. Миграция `broll_meta`.
2. Доработать `fetch-broll` (контекст, alternatives, min_duration).
3. Создать `broll-block-action`.
4. Переписать `BrollPanel` с интерактивом.
5. Доработать чип B-roll в `ScenesPanel`.
6. Прогнать на текущем проекте, убедиться, что нет миганий и темы попадают.
